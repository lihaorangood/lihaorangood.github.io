# 富文本的简单实现

主要依赖于H5新增的contenteditable 属性 可以将之前非input的DOM元素也具有输入和编辑功能 可以应用于div、body等元素上完成富文本编辑 聊天输入框等功能

   <div contenteditable="true"></div>

# 如何实现文字复制粘贴以及 图片粘贴 视频上传

通过监听剪切板的复制，如果需要纯文本粘贴 我们需要进行样式的过滤处理只保留文字， contenteditable的容器回车默认会插入div或者p根据浏览器的默认行为而定 ，如果我们想要有其他的标签替换，可以监听键盘的keydown事件 阻止默认行为， 如果是图片粘贴分为两种，一种是将图片转成base64图片存储在剪切板，一种是网站特殊处理，我们复制时便会生成对应的存储在剪切板中，我们进行粘贴操作时也要进行区分，一般我们看到的富文本比如UEditor的操作按钮和body编辑区域不在同一个html页面中，我们进行图片上传需要用到两个界面（html和iframe嵌套界面）之间的通信 ， postmessage可以完成不同界面的通信，来进行按钮操作界面的目的

如果我们复制粘贴或者进行操作设计的DOM太多，比如生成表格等等操作， 我们可以使用fragement或者Range来进行操作， 减少富文本的重绘， 也可以类似Vue这种采用Object描述文档树的组成， 然后对比 局部和精准更新，减少消耗。

// contenteditable 容器上添加keydown监听
 if (e.keyCode === 13) {   
        // insert 2 br tags (if only one br tag is inserted the cursor won't go to the next line)
        document.execCommand('insertHTML', false, '<br/>');
        // prevent the default behaviour of return key pressed
        return false;
 }

# UEditor 配合 Vue 使用

  • 去官网下载Ueditor的文件
  • 部署在一个端口，Vue默认自带的就有一个启动端口， 放在静态文件存储地点即可
  • npm i vue-ueditor-wrap 这个是再稍微封装的配合Vue使用的容器

  • vue-ueditor-wrap 提供了几个属性和一个hooks函数，分别为
    1. v-model 最终我们得到的html richString 发送给后端存储的部分， 只包括body的innerHTML
    2. config 配置对象
    3. @ready 富文本加载完成的回调函数 会将 富文本的实例作为参数 里面的body属性就是添加了contenteditable的body DOM 对象， 我们可以利用这个body做一些自定义的操作，比如监听复制粘贴，以及不同的粘贴位置做不同的处理等等
    4. @before-init 加载之前做的一些准备， 比如自定义操作按钮 自定义弹窗 等等
