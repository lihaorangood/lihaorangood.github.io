lihaoran 2022-04-02 vue

主要实现了 Vue 的初步实现 但是仍有很多需要完善的地方 比如指令解析 组件缓存优化 组件传值等等 在后面会逐步实现

# Vue3 初体验

# vue3 使用 setup 的原因

使用 Vue2 中的写法 data method watch 总是分开的 不利于代码的查看编写和维护 在 setup 函数中我们可以把相关的内容都放在一起 方便维护,对于使用开发者来说是一个极好的体验， 并且使用compositions API 方便了打包 减少了打包体积 Vue2比如watch nextTick这些无论使用还是不使用都会直接挂载进行打包

# setup 的执行时机

setup 是在初始合并参数之前 也就是之前的 initState 函数之前执行的 所以此时 props data methods watcher computed 都还没有通过代理到组件实例上 所以我们无法访问到这些属性,在 Vue3.0 中 setup 中定义的变量是需要通过 return 返回出去才能在视图中使用,但是考虑到可能忘记返回造成开发的不便 3.2 版本升级解析器并通过 script 标签中加入 setup 属性就可以直接使用 不用返回,SFC组件名称会自动推断 如果我们想要明确的组件名称 我们可以多使用一个script标签 然后将name名称export出去， 在setup中引入的组件不需要使用components注册直接就可使用， 并且defineProps和defineEmits从vue引入可以直接使用

# 基本使用

使用方式基本和Vue2相同 使用形式发生了一些变化 变成了回调函数形式

  • provoide inject 提供了高层级组件向低层级组件直接提供数据的方式 不需要一层层的传递
  • emit 相当于以前的$emit 不过需要在组建中声明接收 然后调用
  • watch , watchEffect watch 监听方式 第一个参数接收监听范围 Function | Array 第二个参数是回调函数 参数和原来形同 watchEffect 会自动收集依赖 所以会立即执行一次
  • expose 配置提供给外部使用组件实例可以使用的方法
  • slot Vue3中插槽只能使用v-slot:xxx这种方式来进行命名
  • Vue3SFC组件可以不使用div等根节点包裹（或者说组件可以有多个根节点）Vue3会自动使用fragement进行包裹 但是这可能会造成一些CSS样式问题 开发中还是建议使用一个div这种根节点进行包裹
<script>
export default {
  name: "cus-button"
}
</script>
<script setup>
import {computed, defineProps, defineEmits} from "vue";
const emits = defineEmits(['click'])
const props = defineProps({
  type: {
    type: String,
    default: 'primary',
    validator(value) {
      const types = ['primary', 'success', 'warning', 'info', 'danger']
      if (types.includes(value)) {
        return true
      } else {
        throw new Error(`this type must be type in ${types}`)
      }
    }
  },
  icon: String,
  loading: {
    type: Boolean,
    default: false
  },
  disable: {
    type: Boolean,
    default: false,
  },
  size: {
    type: String,
    default: 'middle',
    validator(sizeValue) {
      const sizes = ['large', 'middle', 'small']
      if (sizes.includes(sizeValue)) {
        return true
      } else {
        throw new Error(`this type must be type in ${sizeValue}`)
      }
    }
  }
})
const classNames = computed(() => {
  const classes =  ['cus-button', `cus-button-${props.type}`]
  if(props.size !== 'middle'){
    classes.push(`cus-button-${props.size}`)
  }
  return classes
})
const emitClick=() =>{
 emits('click')
}
</script>
<template>
  <button :class="classNames" :disabled="disable" @click="emitClick">
    <span v-if="$slots.default">
      <slot></slot>
    </span>
    <cus-icon class="icon" v-if="icon && !loading" :icon="icon"/>
    <cus-icon class="icon loading" v-if="loading" icon="loading"/>
  </button>
</template>
